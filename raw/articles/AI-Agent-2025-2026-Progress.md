# AI Agent 2025-2026 进展全景

## TL;DR

2025 年是 AI Agent 从理论走向基础设施的决定性转折年。关键进展包括：MCP 协议成为行业标准（9700 万安装）、Agent2Agent 协议发布、91% 工程师已使用 agentic 编码工具。2026 年焦点转向实用化、治理框架和安全挑战。

## 2025 年：Agent 元年

> "2025 was the year the concept became concrete for developers and consumers alike."

### 关键时间线

| 时间 | 事件 |
|---|---|
| 2025.01 | DeepSeek-R1 开源发布，加剧全球 AI 竞争 |
| 2025.02 | GitHub Copilot Agent Mode 宣布 |
| 2025.04 | Google 发布 Agent2Agent 协议 |
| 2025.05 | GitHub Copilot Coding Agent 嵌入 GitHub |
| 2025 中期 | 消费级 Agentic 浏览器涌现（Perplexity Comet, Opera Neon, GPT Atlas）|
| 2025 年末 | Anthropic 和 Google 将协议捐赠给 Linux Foundation |
| 2026.03 | MCP 跨越 9700 万安装里程碑 |

### 核心协议标准

#### Model Context Protocol (MCP)

> "USB-C for AI" - 让 AI Agent 与外部工具通信

- **功能**：连接数据库、搜索引擎、API
- **地位**：从实验标准升级为基础设施
- **里程碑**：2026 年 3 月超 9700 万安装
- **开放治理**：Linux Foundation 接管

#### Agent2Agent Protocol

- **发布**：Google 2025 年 4 月
- **功能**：解决 Agent 间通信问题
- **设计**：与 MCP 协同工作
- **开源**：同样捐赠给 Linux Foundation

> "While MCP focused on how agents use tools, Agent2Agent addressed how agents communicate with each other."

## Agentic 编码现状

### 行业采用率 (2026 年 3 月调查)

- **91%** 工程师在工作中使用 agentic AI 编码工具
- **75%** 在过去 6 个月发布了 AI 部分或主要生成的生产代码

### 主流工具

| 工具 | 特点 |
|---|---|
| Claude Code | Anthropic 的 CLI agentic 助手 |
| Cursor | IDE 集成的 AI 编码 |
| GitHub Copilot | Agent Mode + Coding Agent |
| Kiro (AWS) | Spec-driven 开发 |
| Codex | OpenAI 代码生成 |

### Agentic 编码定义

> "Agentic coding is a software development approach where autonomous AI agents plan, write, test, and modify code with minimal human intervention."

**与传统 AI 助手的区别**：
- 传统：等待用户输入
- Agentic：接收高层指令，自主执行
- 能力：导航文件系统、管理依赖、运行终端命令、自动修复错误

### OpenAI 愿景

Fidji Simo (OpenAI 应用 CEO)：

> "A year from now, answering questions will be the least useful thing AI can do. Instead, we'll have proactive AI assistants constantly running in the background, getting things done for us across the web and the real world."

### 挑战与风险

1. **安全风险**
   - Claude Code 被滥用于网络攻击案例
   - Agent 降低恶意活动门槛

2. **评估标准**
   - 传统基准不适用于组合系统
   - 需要评估过程而非仅结果

3. **系统性问题**
   - 数据中心能源压力
   - 就业置换担忧
   - 监控问题

4. **治理框架**
   - Agentic AI Foundation 成立
   - 需要新的评估标准

## Spec-Driven 开发

新范式：先规格，后编码

> "Instead of coding first and writing docs later, spec-driven development starts with specifications that become the shared source of truth."

**优势**：
- 规格成为共享真相来源
- 为 coding agent 工作流提供结构化流程
- 提高 Agent 输出的可预测性
