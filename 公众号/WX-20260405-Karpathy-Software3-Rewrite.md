# Karpathy 最新演讲：软件正在发生巨变，而这次人人都能当程序员

> "我在 Twitter 上发了 15 年，至今不知道哪条推文会火。'Vibe Coding'这条本来觉得没人会理，结果成了热梗，现在连维基百科词条都有了。"

这是 Karpathy 上个月在 YC AI Startup School 的演讲里开的一个玩笑。

但他讲的内容一点都不是玩笑。

## 一、软件 70 年没变过，现在一年变两次

Karpathy 开场就说：

**"软件在根本层面上已经 70 年没变了。然后在过去几年里，它变了两次。"**

什么意思？

他把软件分成三代：

- **Software 1.0**：你写代码，告诉电脑干啥
- **Software 2.0**：神经网络的权重，不是你写的，是训练出来的
- **Software 3.0**：用自然语言编程——对，就是用英语写程序

第三代才是真正的巨变。

> "令人惊讶的是，我们现在用英语给电脑编程。"

这句话他几年前发在推特上，成了他点赞最高的推文。

## 二、LLM 是新的操作系统，不是聊天机器人

很多人把 ChatGPT 当成一个更聪明的搜索引擎，或者一个能陪你聊天的机器人。

Karpathy 说：不对。

**LLM 本质上是一个操作系统。**

他画了一张图：
- LLM 是 CPU
- 上下文窗口是内存
- 各种工具调用是外设

听起来很抽象？他打了几个比方：

**1. 像 1960 年代的大型机**

那时候计算机太贵，只能放在机房里，大家通过终端分时共享。

现在的 LLM 也一样——算力太贵，只能放在云端，我们都是"瘦客户端"。

**2. 像电网**

OpenAI、Anthropic、Google 这些公司就像发电厂，投入巨资建"AI 电网"，我们按 token 付费用电。

有意思的是，前几天好几个 LLM 同时宕机，全世界的人都干不了活了。

> "这就像智能层面的电压不稳——LLM 一挂，地球就变笨了。"

**3. 像 Windows vs Mac vs Linux**

闭源的 OpenAI、Anthropic、Google，就像 Windows 和 macOS。

开源的 LLAMA 生态，可能会长成 Linux。

现在还早，但趋势已经很明显了。

## 三、LLM 有点像自闭症天才

这是 Karpathy 最有意思的一个观察。

他说 LLM 让他想起电影《雨人》里的达斯汀·霍夫曼——一个能记住整本电话簿的自闭症天才。

LLM 也是这样：
- 能记住 SHA 哈希值
- 能背下来百科全书
- 某些领域碾压人类

但同时：
- 会坚持说 9.11 大于 9.9
- 数不清草莓里有几个 r
- 容易被骗，极度 gullible

> "它可能上一秒是博士，下一秒被小学生的脑筋急转弯难住。"

还有一个更要命的问题：

**LLM 有"顺行性遗忘症"**——就是《记忆碎片》那部电影里的毛病。

你的同事加入公司后，会慢慢了解公司、积累经验、越来越熟练。

LLM 不会。每次对话结束，它就忘了。第二天又是新的一天。

> "想象一下，你的同事每天早上醒来都忘了你是谁，这就是 LLM 的日常。"

## 四、不要造机器人，要造钢铁侠战甲

讲到这里，Karpathy 讲了一个故事。

2013 年，他第一次坐自动驾驶汽车——朋友在 Waymo 工作，带他在 Palo Alto 兜了 30 分钟风。

那次完全没有人工介入。完美。

他当时想：自动驾驶马上就要实现了！

**结果呢？12 年过去了，我们还在做。**

Waymo 的车看起来没人开，但背后还是有大量远程操控和人工干预。

> "所以当我看到'2025 是 Agent 之年'这种说法，我就很担心。这应该是 Agent 的十年，不是一年。"

他的建议是：

**现阶段不要造自主机器人，要造钢铁侠战甲。**

什么意思？

钢铁侠的战甲既能增强托尼的能力，也能在某些场景下自主行动。但关键是——人在回路里。

Cursor 就是一个好例子：
- 你可以按 Tab 只接受一小段补全（你主导）
- 可以 Command+K 改选中的代码（AI 帮忙）
- 可以 Command+I 让它改整个仓库（AI 主导）

这就是"自主程度滑块"——你控制 AI 有多大自由度。

> "与其造酷炫的自主 demo，不如造实用的半自主产品。"

## 五、Vibe Coding：每个人都是程序员

这是 Karpathy 演讲里最爆的部分。

他说他发了条推特讲"Vibe Coding"，本来以为没人理，结果火得一塌糊涂，现在连维基百科都有词条了。

什么是 Vibe Coding？

就是用自然语言"凭感觉"写程序。不用学 5 年编程，周六下午想到一个点子，当天就能做出来。

他自己就这么干过：

**例子 1：iOS App**

他不会 Swift，但花了一天做了一个能跑在 iPhone 上的 App。

> "以前我得先学 5 天 Swift 才能开始。现在当天就跑起来了。"

**例子 2：MenuGen.app**

他去餐厅看不懂菜单（谁能想到 Karpathy 也有这烦恼），于是做了一个 App：拍菜单照片，AI 生成菜品图片。

核心功能一个下午就做完了。

但接下来的一周全在折腾：认证、支付、域名、部署……

> "代码是最简单的部分。真正痛苦的是 DevOps。"

他说这个 App 是他的"重大成本中心"——每个新用户送 5 美元额度，他一直在亏钱。

但关键不是赚钱。关键是：

**以前只有程序员能做的事，现在人人都能做了。**

Tom Wolfe 分享了一个视频：一群小孩在 Vibe Coding。

Karpathy 说：

> "看这个视频怎么会对未来悲观？这些孩子会把 Vibe Coding 当成编程的入门。未来一片光明。"

## 六、给开发者的建议

Karpathy 最后给了几条实操建议：

### 1. 为 Agent 而建

你的产品文档，要让 LLM 能读懂：
- 用 Markdown
- 写清楚 API 怎么用
- 考虑做一个"robots.txt for LLMs"

### 2. 造 GUI，不要只有文本

人类读文本很累，但看 diff 的红绿对比秒懂。

GUI 不是为了好看，是为了让人类能更快验证 AI 的输出。

### 3. 控制自主程度

给你的产品加一个"自主滑块"：
- 低自主：AI 只做建议
- 中自主：AI 执行，人类审批
- 高自主：AI 全权处理

根据任务复杂度，让用户选择信任程度。

### 4. 保持 AI "在绳子上"

Prompt 越具体，验证成功率越高，循环转得越快。

模糊的指令 → AI 乱跑 → 验证失败 → 重来

具体的指令 → AI 精准执行 → 验证通过 → 下一步

> "Prompt 含糊，就等着原地打转吧。"

## 结语

Karpathy 最后说：

> "我们现在大概处于 LLM 的 1960 年代。个人电脑革命还没发生，但已经有人在尝试了——Mac Mini 跑本地 LLM，某种意义上就是早期迹象。"

软件正在重写。

不是修修补补，是从底层重写。

而这一次，你不需要计算机学位才能参与。

你只需要会说话。
