# 知识库健康检查命令

## 用法
```
/health-check
```

## 功能
对知识库进行全面体检，发现并报告问题。

## 检查项目

### 1. 一致性检查
- 同一概念在不同地方定义是否冲突
- 术语使用是否统一

### 2. 完整性检查
- 概念条目是否缺少定义
- 摘要是否缺少关键字段
- 来源引用是否完整

### 3. 孤岛检测
- 入链出链少于 2 的笔记
- 没有被任何其他笔记引用的内容

### 4. 过期检测
- 超过 30 天未更新的概念
- 可能已过时的信息

## 输出报告

存储位置: `outputs/health/Health-YYYYMMDD.md`

```markdown
---
check_date: YYYY-MM-DD
---

# 健康检查报告 - YYYY-MM-DD

## 统计概览
- 总文件数: X
- 概念条目: X
- 摘要文件: X

## 一致性问题
| 概念 | 问题描述 | 涉及文件 |
|---|---|---|

## 完整性问题
| 文件 | 缺失字段 |
|---|---|

## 孤岛笔记
- [[文件名]] - 入链: X, 出链: X

## 建议操作
- [ ] 任务 1
- [ ] 任务 2
```

## 自动修复
部分问题可自动修复：
- 补充缺失的 frontmatter 字段
- 添加双向链接
